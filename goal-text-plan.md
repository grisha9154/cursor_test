# Goal → text plan

Turn a one-line business or product goal into a short, ordered checklist. Each step should be one concrete action you can do next.

## How to use

1. Write your goal in plain language (what the user should get or what you will ship).
2. Break it into 4–8 steps: files to add, UI to build, logic to implement, wiring (events, routes, tests).
3. Order steps so dependencies come first (e.g. create file → build UI → add behavior → connect).

## Example

**Goal:** Create button with text “Open Dialog”

**Text plan:**

1. Create a file with a name that matches the feature (e.g. `open-dialog-button`).
2. Add markup/UI for a simple button labeled “Open Dialog”.
3. Implement a function that opens the dialog (create/show dialog, set state, etc.).
4. Connect the button’s click (or equivalent) to that function.
